- **框架架构**：采用 Next.js App Router 模式，通过 `app/` 目录实现文件系统路由，包含静态展示页（如 `/about`, `/campuses`）及 API 路由（`/api/booking`）。
- **组件化设计**：UI 逻辑解耦至 `components/` 目录，首页由多个业务区块组件（Hero, CoursesSection 等）组合而成；布局由 `app/layout.tsx` 统一控制，注入 Navbar、Footer 及全局样式。
- **数据管理**：采用轻量级静态数据方案，所有业务数据（课程、师资、校区）集中定义在 `lib/data.ts` 中供全栈引用。
- **交互逻辑**：预约表单（BookingForm）作为客户端组件（"use client"），通过 Fetch API 与后端 Route Handler 通信，实现无刷新提交。
